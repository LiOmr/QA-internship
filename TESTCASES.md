# E2E тест-кейсы для таск-трекера

## Общие допущения
- Пользователь заходит на страницу таск-трекера (авторизация не требуется).


## TC-01 - Создание задачи (позитивный сценарий)

Цель: проверить, что пользователь может создать задачу, заполнив обязательные поля, и задача появляется в списке.

Предусловия:
- Открыта страница "Все задачи".

Тестовые данные:
- Название: `E2E Task <timestamp>`
- Описание: `Создано автотестом`
- Проект: любое доступное значение
- Приоритет: любое доступное значение
- Статус: `Backlog` (по умолчанию) или любое доступное значение
- Исполнитель: любое доступное значение

Шаги:
1. Нажать кнопку "СОЗДАТЬ ЗАДАЧУ" в верхней панели.
2. Убедиться, что открылось модальное окно "Создание задачи".
3. Ввести в поле "Название" значение `E2E Task <timestamp>`.
4. Ввести в поле "Описание" значение `Создано автотестом`.
5. В выпадающем списке "Проект" выбрать значение.
6. В выпадающем списке "Приоритет" выбрать значение.
7. В поле "Статус" оставить по умолчанию `Backlog`.
8. В выпадающем списке "Исполнитель" выбрать значение.
9. Убедиться, что кнопка "СОЗДАТЬ" стала активной.
10. Нажать кнопку "СОЗДАТЬ".

Ожидаемый результат:
- Модальное окно закрывается.
- В списке задач появляется новая задача с названием `E2E Task <timestamp>`.
- У задачи отображаются связанные данные.


## TC-02 - Создание задачи без названия (негативный сценарий)

Цель: проверить, что нельзя создать задачу без заполнения обязательного поля "Название".

Предусловия:
- Открыта страница "Все задачи".

Шаги:
1. Нажать кнопку "СОЗДАТЬ ЗАДАЧУ".
2. Убедиться, что открылось модальное окно "Создание задачи".
3. Оставить поле "Название" пустым.
4. Заполнить остальные обязательные поля:
   - выбрать "Проект"
   - выбрать "Приоритет"
   - статус оставить по умолчанию Backlog
   - выбрать "Исполнитель"
5. Проверить состояние кнопки "СОЗДАТЬ".
6. Нажать "СОЗДАТЬ", если кнопка активна.

Ожидаемый результат:
- Задача не создается.
- Кнопка "СОЗДАТЬ" остается неактивной, так как поле "Название" пустое.  


## TC-03 - Открытие карточки задачи (модальное окно редактирования)

Цель: проверить, что при клике по задаче открывается карточка задачи модальное окно "Редактирование задачи" и данные в ней соответствуют выбранной задаче.

Предусловия:
- Открыта страница "Все задачи".
- В списке есть хотя бы одна задача.

Тестовые данные:
- Название задачи: `E2E Task <timestamp>` (или любое существующее из списка)

Шаги:
1. На странице "Все задачи" найти в списке задачу с нужным названием.
2. Нажать на карточку задачи.
3. Дождаться открытия модального окна.

Ожидаемый результат:
- Открывается модальное окно с заголовком "Редактирование задачи".
- В поле "Название" отображается название выбранной задачи.
- Отображаются и доступны поля задачи: Описание, Проект, Приоритет, Статус, Исполнитель.
- В нижней части модального окна отображаются элементы управления: кнопка "ОБНОВИТЬ" и кнопка "ПЕРЕЙТИ НА ДОСКУ".



## TC-04 — Поиск задачи (позитивный сценарий)

Цель: проверить, что список задач фильтруется по введенному тексту в поле "Поиск" и отображаются подходящие задачи.

Предусловия:
- Открыта страница "Все задачи".
- В системе существует задача с названием `E2E Task <timestamp>`.

Шаги:
1. В блоке "Список задач" нажать на строку поиска.
2. Ввести в поле поиска текст `E2E Task <timestamp>`.
3. Дождаться обновления списка задач.

Ожидаемый результат:
- Список задач фильтруется в соответствии с введенным запросом.
- В результатах отображается задача с названием `E2E Task <timestamp>`.
- Среди отображаемых задач нет задач, не соответствующих запросу.


## TC-05 — Поиск задачи (негативный сценарий)

Цель: проверить, что при вводе запроса, по которому нет совпадений, система корректно показывает пустой результат поиска.

Предусловия:
- Открыта страница "Все задачи".

Тестовые данные:
- Поисковый запрос: `E2E_DOES_NOT_EXIST_<timestamp>` 

Шаги:
1. Нажать на поле поиска.
2. Ввести значение `E2E_DOES_NOT_EXIST_<timestamp>`.
3. Дождаться обновления списка задач.

Ожидаемый результат:
- В блоке результатов отображается сообщение "Задачи не найдены".
- Список задач отсутствует.
- В поле поиска отображается кнопка очистки.
- При нажатии на кнопку очистки поле поиска очищается, и список задач возвращается к исходному состоянию.



## TC-06 - Переход на доску проекта

Цель: проверить, что пользователь может перейти на доску выбранного проекта и увидеть колонки со статусами и задачи.

Предусловия:
- Доступен раздел Проекты и у проектов есть действие "ПЕРЕЙТИ К ДОСКЕ".
- В выбранном проекте есть хотя бы одна задача (или допускается пустая доска).

Шаги:
1. В верхнем меню нажать "ПРОЕКТЫ".
2. На странице "Проекты" найти любой проект.
3. Нажать у выбранного проекта кнопку "ПЕРЕЙТИ К ДОСКЕ".

Ожидаемый результат:
- Открывается страница доски выбранного проекта.
- На доске отображаются колонки со статусами: To Do, In Progress, Done.
- В колонках отображаются карточки задач (если в проекте есть задачи).
